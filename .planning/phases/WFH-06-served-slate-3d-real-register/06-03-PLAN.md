<objective>
Port REAL-03 — image billboards with single-fetch persistence — from
`cp/sprite_manager.js` into `fe/projector.mjs`. Image-bearing chunks render as
`THREE.Sprite` billboards whose textures load through the exact ordered cache chain
(in-memory `Map` → IndexedDB blob → `/api/image_proxy` fetch → direct fetch),
sharing one `THREE.Texture` per URL, with the `X-Image-Proxy-Note` transparent-PNG
fallback never written to the cache as a success. Collider spacing
(`NODE_RADIUS = 0.9`) is shared with text billboards (Plan 01's force step).

Purpose: REAL-03 makes the served projector show real scanned imagery instead of
flat HSV points for image chunks, with cross-session persistence (reusing the legacy
`wfh_texture_cache` DB name so a returning user's cache carries over to `fe/` with
zero migration). The `X-Image-Proxy-Note` guard prevents cache-poisoning (Pitfall 3).
Output: `projector.mjs` gains the IndexedDB helpers, `loadAndCacheImage(url)` fetch
chain, and `spawnImageBillboards()` (THREE.Sprite, aspect-corrected, shared collider
radius); `editor.html` exposes a `__mm_rerender` + image test hook; `projector.spec.js`
gains the `image` e2e block.
</objective>
