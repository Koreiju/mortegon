<objective>
Port REAL-04 — the solid, headless 2D↔3D link arrow — from
`cp/concept_graph.js::_drawConcept3DLinks` into the served surface. Every pinned
panel gets a `data-3d-node-id` attribute (NEW authorship — confirmed absent from
`fe/` today); a new `#link-layer` SVG overlay is added to `editor.html`; and a
per-animate-frame drawer projects each tagged node's CURRENT (post-force-step) world
position through the camera and draws a solid `#ffd700` `stroke-width:2` line with NO
`marker-end` and NO `stroke-dasharray` from the panel to that screen point, hiding it
when the node is off-frustum. The arrow drives nothing — it is pure derived render
state — and must keep the existing `black_slate.spec` no-dotted gate green.

Purpose: REAL-04 visually ties each 2D pinned panel to its 3D node, the last of the
four ported Real-register features. It reuses the projector's existing `project()`
helper (the SAME function the halo ray-transport uses) and the post-force-step
positions from Plans 01/03 so the line tracks the moving node. The solid/headless/
no-dasharray styling is a hard project-wide rule (CLAUDE.md Forbidden Concepts).
Output: `editor.html` gains the `#link-layer` SVG host, sets `data-3d-node-id` at
panel mount, and calls `drawConcept3DLinks` each frame; `projector.mjs` gains
`drawConcept3DLinks(svgHost)`; `projector.spec.js` gains the `arrow` e2e block.
</objective>
